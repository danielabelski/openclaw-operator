# OpenClaw Operator Console --- Visual Specification

## Overall Frame

The interface must render as a mechanical industrial command console
rather than a standard web dashboard. The entire page should resemble a
single embedded hardware control unit mounted inside a machine chassis.

The UI must appear heavy, metallic, machined, and bolted together, as if
it were a physical console rather than a flat webpage.

### Frame Characteristics

-   Brushed dark steel texture
-   Visible bolts in each corner
-   Layered metal plates
-   Recessed grooves and seams
-   Orange industrial indicator lights embedded in the side rails

The frame must visually communicate that the console is installed inside
a machine housing.

The page should never look borderless. The frame must occupy the entire
outer edge of the screen.

------------------------------------------------------------------------

## Side Rails

Both left and right sides contain vertical mechanical rails.

These rails include: - Metallic pipes and conduit - Circular mechanical
knobs - Glowing amber status lights - Recessed industrial panels

The rails must appear like machine infrastructure supporting the console
module.

------------------------------------------------------------------------

## Header Bar

The top section is a solid mechanical header bar.

Characteristics: - Thick brushed metal plate - Engraved OpenClaw logo
area - Mechanical seam lines across the metal - Slightly beveled edges

### Left Side

Large red lobster mascot icon rendered as a glossy 3D emblem mounted
onto the metal panel.

Next to it: **OPENCLAW** in large uppercase industrial sans‑serif
typography.

------------------------------------------------------------------------

## Top Navigation Tabs

Below the title is a metal tab navigation strip.

Tabs: - Overview - Tasks - Approvals - Agents - Governance - System
Health

Each tab appears as: - Metallic plate button - Inset text label -
Glowing highlight when active - Slightly raised hardware feel

The active tab appears illuminated or engaged.

------------------------------------------------------------------------

## Top Right Controls

### Public Proof Button

Rectangular dark metallic button labeled **Public Proof** with beveled
edges.

### Operator Profile

Circular avatar image of the operator placed at the far right of the
header.

------------------------------------------------------------------------

# First Status Panel Row

Four large machine panels appear directly under the header.

Panels: - System Health - Persistence - Task Queues - Approval Status

Each panel contains: - Metallic frame - Recessed display area - Status
icon - Title label

------------------------------------------------------------------------

Label: **System Health**

Displayed on glowing green rectangular status field.

------------------------------------------------------------------------

## Persistence Panel

Indicator: Orange warning triangle.

------------------------------------------------------------------------

# Middle Section Panels

Two wide panels appear below the first row.

Left panel: **Governance Summary**\
Right panel: **Persistence Status**

Panels must appear heavier and larger than the top cards.

------------------------------------------------------------------------

## Governance Summary Panel

Rows displayed with green check icons.

Entries:

-   Approval Backlog: 2 Tasks
-   Retry Recovery: 1 Job
-   Milestone Backlog: 4 Targets
-   Demand Backlog: 3 Requests

Rows should resemble industrial monitoring list items.

------------------------------------------------------------------------

## Persistence Status Panel

Subsystem health rows with colored status dots.

Entries:

-   Database Health: Partial (red)
-   Queue Storage: Stable (green)
-   Recall Memory: Degraded (red)
-   Knowledge Store: Metadata Only (yellow)

------------------------------------------------------------------------

# Activity Row Design

Rows resemble system log entries.

Each row includes: - Colored status indicator - Bold task name - Status
badge - Agent label - Timestamp

Rows are placed inside metal framed horizontal containers.

------------------------------------------------------------------------

# Right‑Side Control Hardware

On the far right side of the activity panel:

-   Two circular mechanical knobs
-   Vertical stack of amber indicator lights

These reinforce the machine console aesthetic.

------------------------------------------------------------------------

# Color System

Green → system healthy\
Amber → warning\
Red → failure or degraded\
Blue → informational

Avoid neon or bright consumer UI colors.

------------------------------------------------------------------------

# Lighting

Panels use subtle glow effects. Metal surfaces reflect dim industrial
lighting.

------------------------------------------------------------------------

# Texture

Background materials:

-   Brushed steel
-   Gunmetal gray
-   Dark graphite

Panels must appear machined rather than flat.

------------------------------------------------------------------------

# Layout Rules

All elements align to a rigid mechanical grid.

Panels should appear embedded in a rack system.

Spacing should feel engineered rather than airy.

------------------------------------------------------------------------

# UX Philosophy

The interface should feel like operating:

-   infrastructure
-   a control system
-   a machine console

The UI must never resemble a marketing dashboard or SaaS analytics
product.

# USER - Who I Serve

## My User

Development teams who need confidence that code is ready to ship.

## Primary Use Cases

1. **Pre-Deployment Verification**
   - User need: Run full test suite before shipping
   - My role: Execute tests, report results
   - Success metric: All tests pass

2. **Regression Detection**
   - User need: Catch broken tests before merge
   - My role: Run tests on every commit
   - Success metric: Zero broken tests in main

## User Expectations

- **Speed**: Full test suite < 5 minutes
- **Accuracy**: Test reports are correct (no false positives)
- **Coverage**: All test types run (unit, integration, e2e)
- **Clarity**: Failed tests have debugging guidance

## Scheduling

- **Frequency**: On-demand + continuous integration
- **Hours**: Available 24/7
- **SLA**: 5 minutes for full test suite
